# 播客转录稿编译为微信公众号文章

一个 Claude Skill，帮你把英文播客或访谈的语音转录稿，自动编译成排版规范的中文微信公众号文章，并直接输出可用的 Word 文档（.docx）。

---

## 这个 Skill 能做什么

- 识别转录稿中的说话人，区分主持人与嘉宾
- 自动修正语音转录中常见的英文专有名词识别错误（产品名、人名、技术术语等）
- 将对话内容翻译成流畅的中文，风格接近科技媒体的专业编译
- 按内容逻辑划分章节，生成带编号的章节标题
- 输出完整排版的 .docx 文件，包含：
  - 多个备选标题
  - 导语金句
  - 带边框的引言模块
  - 带颜色标签的对话正文（主持人 / 嘉宾视觉区分）
  - 专有名词解释备注框

---

## 适合什么场景

- 科技播客的中文编译（访谈、圆桌、深度对话等）
- 英文会议或内部访谈录音的整理
- 任何需要把英文对话转成中文可读文章的场景

---

## 如何使用

**1. 准备转录稿**

将音频上传至语音转录工具（如飞书、Whisper、Otter.ai 等），导出文字稿。说话人标注格式通常为"说话人1""说话人2"，无需手动整理。

**2. 把 SKILL.md 提供给 Claude**

在对话开始时，将 `SKILL.md` 的内容粘贴给 Claude，或作为文件上传，告诉它按照这份 Skill 来处理接下来的转录稿。

**3. 上传转录稿**

将导出的文字稿上传或粘贴给 Claude。

**4. 确认结构预览**

Claude 会先输出章节结构和语气样本，等你确认后再生成完整文档，不会直接一次性输出，方便你在早期调整方向。

**5. 下载 Word 文档**

确认后，Claude 生成完整 .docx 文件并提供下载链接，直接可用于发布或进一步编辑。

---

## 输出示例

文档结构如下：

```
备选标题（4-5 个）
摘要金句（2-3 句）
引言（200-300 字 + 核心观点列表）

01. 章节标题
    [主持人姓名]：……
    [嘉宾姓名]：……

02. 章节标题
    ...
```

对话段落带颜色标签（主持人橘色底白字，嘉宾浅橘底橘字），金句加粗，专有名词附备注框。

---

## 依赖环境

本 Skill 的技术实现依赖 Claude 的代码执行能力，使用 `docx` npm 包生成 Word 文件。在 Claude.ai 的标准环境中可直接运行，无需额外配置。
